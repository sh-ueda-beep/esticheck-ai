## Why

oo省向け契約見積業務では、10〜30シートのExcelブックを手作業で修正・検証しており、#REF!等の数式エラー頻発・チェック工数の増大・属人化が課題となっている。AIエージェントによるExcel構造解析・エラー検出・品質スコアリングを自動化し、見積品質の向上とチェック工数の削減を実現する（POC Phase 1）。

## What Changes

- 新しいReactページ群を追加: Excelアップロード → 分析中 → ダッシュボード（概要/検出事項/AI提案/シート詳細）
- SheetJSによるブラウザ内Excel解析エンジンを実装（サーバー送信なし）
- 6ステップのAIエージェントパイプラインをルールベースで実装（構造解析 → 名前定義 → 費目認識 → 整合性チェック → 異常値検出 → 提案生成）
- 品質スコアリング（100点満点）と優先度付き改善提案の自動生成
- 費目構造の自動認識と原価構成の可視化（棒グラフ）
- 既存のチャットUIとは独立した新しいアプリケーションルートとして追加

## Capabilities

### New Capabilities

- `excel-upload`: .xlsxファイルのブラウザアップロードとSheetJSによるWorkbookオブジェクト変換（最大50MB）
- `sheet-analysis`: 各シートの行列数・数式数・結合セル数・クロスシート参照・複雑度の算出
- `formula-error-detection`: #REF!等の数式エラー、外部ブック参照、手動検証式（IF(...,"計算間違い!!")）の検出
- `cost-structure-recognition`: 費目キーワード（直材/加工/設計/試験/出張/梱包/雑費等）の自動認識と金額紐付け
- `anomaly-detection`: 同一カテゴリ費目間の統計的異常値検出（平均±2σ）
- `quality-scoring`: 検出結果を統合した100点満点の品質スコア算出と改善提案生成
- `analysis-dashboard`: 品質スコアゲージ・統計カード・シートマップ・費目チャート・検出事項一覧・AI提案のダッシュボードUI

### Modified Capabilities

（既存スペックなし — 新規プロジェクト機能のため変更対象なし）

## Impact

- **依存関係の追加**: `xlsx`（SheetJS）、`recharts`または`chart.js`（グラフ描画）、`lodash`（統計処理）
- **フロントエンド**: 新しいルート/ページコンポーネント群の追加。既存の`App.tsx`（チャットUI）は変更しない
- **ルーティング**: `react-router-dom`の導入が必要（チャットUI ↔ 見積チェッカーの切り替え）
- **ビルド**: フロントエンドのみの変更。バックエンド（AgentCore/CDK）への影響なし
- **セキュリティ**: Excelはブラウザ内のみで処理しサーバーに送信しない（POCフェーズ）
